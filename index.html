mpz

Music player for big local collections

Directory tree is your music library

Features a 3-column UI: directory tree (aka library), playlists and tracks from the selected playlist. Middle-click on a folder or track in the directory view and it will become a playlist, and its content will be loaded into the tracks view. Create multiple playlists from folders in your library and switch between them quickly. Similar to "Album list" in Foobar2000.

Native C++/Qt - fast and responsive UI

Built with C++ and Qt — lightweight and fast. Open source and cross-platform. Uses audio codecs installed on your system (GStreamer through QtMultimedia on Linux). Follows your desktop theme. The "z" in the name stands for "zero bullshit", whatever that means.

mpd client mode

Supports mpd. You can add an mpd server as a library. This is an experimental feature. There is an mpz-mpd impedance mismatch — see more in the README.

Radio streams, hotkeys, playback log and more

Supports radio streams in pls and m3u playlists. Supports CUE. Has a playback log which is helpful for radio streams — never miss a track you liked. Supports MPRIS on Linux for remote playback control, for example via KDE Connect. Got feedback? Feature request? Found a bug? Use the built-in feedback form.

Already using the old Open Build Service repositories?

The previous Open Build Service repositories still exist and existing installations will keep working, but existing users are also encouraged to switch — future packaging effort is focused on omnipackage. The new repositories are signed with different GPG keys, so switching means removing the old repository and adding the new one (the install pages linked above walk through this).

Note: on Debian-based distros (Debian, Ubuntu) the OBS builds were always built against Qt5, even on releases that ship a recent enough Qt6 — making OBS produce working .deb packages with Qt6 turned out to be very painful. The omnipackage builds use Qt6 where the distro supports it.

Not an attempt to clone Foobar2000

Inspired by Foobar2000 and especially its "Album list" feature, mpz is not meant to be yet another cross-platform Foobar clone. It's simply not feasible, and not needed by the author. Instead, mpz has all the features the author has been using for many years (quite literally the same unchanged Foobar2000 config since 2005 or so). You may find it useful too, or hate it for the lack of expected features. The author would love to hear from you — just use the built-in feedback form or GitHub issues.
